1. 프로젝트 개요 
1.1 프로젝트명 
밥터뷰 (BabTerview) — 로컬 맛집 체험단 · 리뷰 플랫폼 
1.2 프로젝트 배경 및 목적 
로컬 음식점 사업자와 일반 리뷰어를 직접 연결하는 모바일 웹 플랫폼 구축 
• Problem (기존의 한계) 
• 소상공인: 기존 플랫폼은 대형 프랜차이즈 위주로 운영되어 소규모 매장의 
접근성과 홍보 기회가 부족함. 
• 리뷰어: 체험단 정보가 여러 채널에 분산되어 있어 거주지 주변의 로컬 맛
집 탐색이 번거로움. 
• Solution (밥터뷰의 해결책) 
• 사업자는 직관적인 대시보드를 통해 쉽게 체험단을 모집 및 관리하고, 리
뷰어는 위치 기반으로 가까운 맛집을 즉시 발견·체험·리뷰할 수 있는 통합 
환경 제공.

샵, 세션 복원

2.3 데이터 흐름 및 핵심 전략 
      핵심 기술 성과: 이중 저장 전략(Dual Storage) & 오프라인 퍼스트(Offline-first) 
네트워크 장애 시에도 앱이 정상 작동하도록 설계하여 사용자 경험(UX)을 극대화했습니
다. 
1. 사용자 액션 → Handler 함수 → State 업데이트 → UI 리렌더링 
2. 이중 저장: localStorage에 즉시 저장 후, Supabase API로 백그라운드 비동기 전송 
3. Graceful Fallback: 서버 요청 실패 시에도 에러 없이 localStorage 데이터를 유지
하여 앱 중단 방지

4. 구현 내용 
4.1 리뷰어 기능 
• 위치 기반 탐색: Geolocation API와 Haversine 공식을 활용한 거리 계산. 5가지 정
렬 옵션 및 14개 카테고리 필터링. 
• 신청/취소 및 리뷰: 이메일 기준 중복 신청 방지, 실시간 모집 현황 업데이트. 장/
단점/개선점이 구조화된 리뷰 시스템 및 이미지 업로드(최대 5장) 지원. 
• 게미피케이션: 포인트 기반 5단계 레벨업 시스템(새싹 → 전문 평가단)으로 사용
자 참여 유도. 
4.2 사업자 기능 
• 체험단 및 신청자 관리: 상품 등록/삭제 시 연관 신청 내역 자동 정리. 신청자 목
록(레벨, 연락처 등) 확인 및 선정 시 자동 알림 발송. 
• 통계 대시보드: Recharts를 활용하여 기간별 핵심 지표(총 신청자, 리뷰 완료율, 평
균 모집률 등) 추이 시각화. 
4.3 공통 기능 및 UI/UX 
• 세션 & 알림: 토큰 기반 세션 복원으로 새로고침 시 로그인 유지. 미읽음 배지를 포함한 직관적인 알림 시스템. 
• UI/UX: 모바일 퍼스트(max-w-md), 따뜻한 톤의 컬러 팔레트(그린/크림/오렌지), 
마이크로 인터랙션(Shimmer 효과, 하트 애니메이션) 적용.

6.2 용어 설명 용어 설명 
SPA (Single Page 
Application) 
HTML 파일 하나에서 JavaScript로 페이지를 동적으로 전환하는 
웹 애플리케이션 방식 
컴포넌트 (Component) UI를 독립적으로 재사용 가능한 조각으로 나눈 것 
Hook React 함수형 컴포넌트에서 상태나 생명주기를 관리하는 함수 
(useState, useEffect 등) 
useMemo 계산 비용이 큰 값을 메모이제이션하여 불필요한 재계산을 방
지하는 Hook 
JWT (JSON Web Token) 서버-클라이언트 간 인증 정보를 안전하게 전달하는 토큰 형식 
Edge Function CDN 엣지 서버에서 실행되는 서버리스 함수 (빠른 응답 속도) 
localStorage 브라우저에 데이터를 영구 저장하는 Web Storage API 
Haversine 공식 지구 표면의 두 지점 사이의 거리를 위도/경도로 계산하는 수
학 공식 
PWA (Progressive Web 
App) 
웹 기술로 네이티브 앱 수준의 사용자 경험을 제공하는 웹 앱 
Prop Drilling 부모→자식으로 여러 단계를 거쳐 props를 전달하는 패턴 (비
효율적) 
Dead Code 어디에서도 호출/참조되지 않는 불필요한 코드 
Graceful Fallback 오류 발생 시 앱이 중단되지 않고 대체 동작을 수행하는 설계
